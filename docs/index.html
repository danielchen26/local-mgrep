Free local semantic code search

Find the right code by meaning, without sending your repo to the cloud.

local-mgrep is a local-first semantic search CLI for developers and coding agents. It indexes a repository locally, retrieves compact source-cited snippets, and returns context that agents can use before expensive file reads.

The project is intentionally positioned as a local version of the semantic-search workflow: no hosted workspace, no paid API key, no remote source upload, and no cloud vector store required for the core search loop.

Quickstart

Install, index, search.

pip install local-mgrep
ollama pull mxbai-embed-large

mgrep index /path/to/repo --reset
mgrep search "where is token refresh implemented?" -m 10 --json

The default path is deliberately short: install the CLI, pull a local embedding model, build a local SQLite index, then ask repository questions in natural language. Use --json when a coding agent or script will consume the result.

Two install paths.

Recommended workflow

Use semantic retrieval as the first pass, then read files deeply.

local-mgrep is not meant to replace reading code. It is meant to make the first move precise: locate the right files and line ranges before a person or agent spends context on full-file reads.

01

Index the repository locally

Run mgrep index once per repository, or use watch mode while editing. The index stores chunks, vectors, paths, and line ranges in SQLite.

02

Ask by intent, not just by token

Search for behavior such as “where is token refresh implemented?” even when the exact identifier is unknown.

03

Inspect cited snippets

Each result includes file path, line range, language, score, and snippet so you can verify provenance before changing code.

04

Hand compact context to an agent

Use --json for machine-readable retrieval packs, then read only the files that the ranked snippets justify.

Why it exists

Agents need better first context.

Coding agents often start by reading broad files or repeating exact searches. local-mgrep gives them a compact ranked context pack: path, line range, language, score, and snippet. Humans can inspect it; agents can cite it; scripts can consume it as JSON.

Without local-mgrep

  • Repeated grep searches
  • Large file reads
  • More context tokens
  • Harder provenance tracking

With local-mgrep

  • One semantic retrieval pass
  • Source-cited snippets
  • Diversified top-k context
  • Stable JSON for agents

Local privacy model

No hosted account. No cloud index. No source upload.

The core workflow runs on your workstation: source scanning, chunking, embeddings, SQLite storage, ranking, result diversification, and optional answer synthesis through local Ollama models.

Search behavior

Semantic-first, practical at ranking time.

Pure vector similarity is good for intent, but code search also needs exact identifiers, paths, and repeated-result control. The ranking pipeline keeps semantic search as the base signal and adds local deterministic adjustments for developer workflows.

  1. Embed query locally. Search intent is converted through the configured Ollama embedding model.
  2. Score vectors. NumPy vector scoring ranks chunks by semantic similarity.
  3. Boost exact code terms. Lexical reranking helps queries with identifiers and path terms.
  4. Deduplicate spans. Logical duplicate source spans are removed.
  5. Diversify final top-k. Repeated chunks from the same file are capped before filling remaining slots.

Answer & agentic modes

Optional local synthesis.

Answer mode synthesizes from retrieved snippets. Agentic mode uses a local model to split broad questions into bounded subqueries.

Answer

mgrep search "how is indexing cleaned?" --answer

Agentic

mgrep search "token lifecycle" --agentic --max-subqueries 3

CLI reference

Commands and flags.

The CLI is organized around four operations: build the index, retrieve context, inspect index state, and keep the index updated while files change.

mgrep index [PATH] [--reset] [--incremental/--full]

Build or refresh the SQLite vector index.

mgrep search QUERY [OPTIONS]

Retrieve semantic results. Supports -m, --json, --answer, --language, --include, --exclude, --semantic-only, and --agentic.

mgrep stats

Show total chunks and indexed files.

mgrep watch [PATH] --interval 5

Refresh the index while the repository changes.

Comparisons

What local-mgrep is, and what it intentionally is not.

The design is optimized for developers who want the semantic-search workflow locally. The tradeoff is intentional: less hosted convenience, more privacy, reproducibility, and zero required usage fee for core search.

Dimensionlocal-mgrepAlternative behavior
Cost modelFree to run locally after installing Ollama and the CLI.Hosted systems can require accounts, API keys, usage billing, or cloud storage.
PrivacySource files, vectors, and SQLite index stay on the workstation.Cloud semantic search may upload source, embeddings, metadata, or query logs.
Search styleNatural-language intent search with line-level snippets and JSON output.Plain grep is exact-token only unless an agent performs repeated searches.
RankingHybrid semantic score, lexical boost, deduplication, and per-file diversification.Basic grep gives matches, not a ranked semantic context pack.
Agent useStable JSON gives agents compact, cited context before full-file reads.Manual search often causes repeated broad reads and weaker provenance.
TradeoffRequires local model setup and local indexing time.Hosted systems may be more turnkey but reduce local control.

Limitations

What not to overclaim.

  • The benchmark is deterministic local context gathering, not provider billing data.
  • Answer quality still requires a full rubric-based agent benchmark.
  • Cloud-grade reranking is intentionally absent; future work can add local rerankers.
  • Hosted accounts, cloud stores, organization switching, and paid web search are intentionally out of scope.

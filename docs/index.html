Free local semantic code search

Find the right code by meaning, without sending your repo to the cloud.

`local-mgrep` is a local-first semantic search CLI for developers and coding agents. It indexes a repository locally, retrieves compact source-cited snippets, and returns context that agents can use before expensive file reads.

Quickstart

Install, index, search.

pip install local-mgrep
ollama pull mxbai-embed-large

mgrep index /path/to/repo --reset
mgrep search "where is token refresh implemented?" -m 10 --json

Installation

Two install paths.

Use PyPI for normal usage or source install for development.

From PyPI

pip install local-mgrep

From source

git clone https://github.com/danielchen26/local-mgrep.git
cd local-mgrep
python3 -m venv .venv
source .venv/bin/activate
pip install -e .

Why it exists

Agents need better first context.

Coding agents often start by reading broad files or repeating exact searches. `local-mgrep` gives them a compact ranked context pack: path, line range, language, score, and snippet. Humans can inspect it; agents can cite it; scripts can consume it as JSON.

Without local-mgrep

  • Repeated grep searches
  • Large file reads
  • More context tokens
  • Harder provenance tracking

With local-mgrep

  • One semantic retrieval pass
  • Source-cited snippets
  • Diversified top-k context
  • Stable JSON for agents

Local privacy model

No hosted account. No cloud index. No source upload.

The core workflow runs on your workstation: source scanning, chunking, embeddings, SQLite storage, ranking, result diversification, and optional answer synthesis through local Ollama models.

Agent context

Designed for machine-readable retrieval.

mgrep search "where is the SQLite schema initialized?" --json -m 10

The JSON shape is stable for agents and scripts:

[
  {
    "path": "local_mgrep/src/storage.py",
    "start_line": 31,
    "end_line": 47,
    "language": "python",
    "score": 0.824,
    "snippet": "CREATE TABLE IF NOT EXISTS chunks ..."
  }
]

Search behavior

Semantic-first, practical at ranking time.

  1. Embed query locally. Search intent is converted through the configured Ollama embedding model.
  2. Score vectors. NumPy vector scoring ranks chunks by semantic similarity.
  3. Boost exact code terms. Lexical reranking helps queries with identifiers and path terms.
  4. Deduplicate spans. Logical duplicate source spans are removed.
  5. Diversify final top-k. Repeated chunks from the same file are capped before filling remaining slots.

Answer & agentic modes

Optional local synthesis.

Answer mode synthesizes from retrieved snippets. Agentic mode uses a local model to split broad questions into bounded subqueries.

Answer

mgrep search "how is indexing cleaned?" --answer

Agentic

mgrep search "token lifecycle" --agentic --max-subqueries 3

CLI reference

Commands and flags.

mgrep index [PATH] [--reset] [--incremental/--full]

Build or refresh the SQLite vector index.

mgrep search QUERY [OPTIONS]

Retrieve semantic results. Supports -m, --json, --answer, --language, --include, --exclude, --semantic-only, and --agentic.

mgrep stats

Show total chunks and indexed files.

mgrep watch [PATH] --interval 5

Refresh the index while the repository changes.

Benchmark evidence

Reproducible local benchmark, not billing data.

At top-k 10 on the deterministic repository navigation benchmark, local-mgrep reaches 30/30 expected-file recall with 2.00× estimated total-token reduction and 2.90× context-token reduction versus a grep-agent simulation. 

.venv/bin/python benchmarks/agent_context_benchmark.py --top-k 10 --summary-only

Limitations

What not to overclaim.

  • The benchmark is deterministic local context gathering, not provider billing data.
  • Answer quality still requires a full rubric-based agent benchmark.
  • Cloud-grade reranking is intentionally absent; future work can add local rerankers.
  • Hosted accounts, cloud stores, organization switching, and paid web search are intentionally out of scope.
